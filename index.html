Architecture Decision Records

Key engineering decisions

Every non-obvious decision was documented with context, trade-offs, and consequences. Six representative decisions from twenty total ADRs.

ADR-002Critical
Distribution

ESM uses .js, CJS uses .cjs extension

Problem

With "type": "module" in package.json, Node.js treats all .js files as ESM. A suri-ui.cjs.js file would break require() in legacy bundlers.

DecisionfileName: (format) => format === "cjs" ? "suri-ui.cjs" : "suri-ui.js"
Impact

Next.js App Router and Webpack 4 consumers can require() the library without configuration.

ADR-003A11y
Accessibility

Loading state uses aria-disabled, not native disabled

Problem

Native disabled removes the element from the tab order. During a 3-second payment submission, the user loses focus context and cannot understand why they cannot proceed.

Decisionaria-disabled="true" + aria-busy="true" + onClick preventDefault. Native disabled only for truly unavailable actions.
Impact

Keyboard users can Tab to a loading button and hear "busy" from their screen reader.

ADR-005Architecture
Architecture

Design tokens imported once in src/index.ts

Problem

@importing tokens inside each .module.css causes Vite to inject the CSS custom properties N times in the consumer bundle — once per component imported.

DecisionSingle import in src/index.ts. Never inside .module.css files.
Impact

Zero CSS duplication regardless of how many components the consumer imports.

ADR-012A11y
Accessibility

aria-live region always mounted, never conditionally rendered

Problem

If the error span is created with content already inside, the browser has no chance to register the live region. The announcement never fires.

DecisionAlways render the description span. Use CSS :empty { display: none } to hide it visually when empty.
Impact

Dynamic form validation errors are reliably announced by VoiceOver and NVDA.

ADR-013Architecture
TypeScript

Polymorphic as prop uses bounded union, not full generics

Problem

Full polymorphic components with ElementType generics require explicit casts that TypeScript strict rejects when combined with forwardRef.

Decisiontype CardElement = "div" | "article" | "section" | "main". Covers 95% of real cases. Full polymorphism documented for v2.0.
Impact

Clean TypeScript without workarounds. Consumer gets autocomplete for the four valid elements.

ADR-020Trade-off
UX Decision

Select uses native <select>, not a custom listbox

Problem

A correctly implemented custom listbox (keyboard nav, ARIA, focus management) is weeks of work. Most implementations ship with subtle bugs that degrade the experience for keyboard and screen reader users.

DecisionNative <select> with appearance: none and SVG chevron via data URI. Activates the native picker on iOS/Android.
Impact

Mobile users get the OS picker. Keyboard and screen reader support is guaranteed by the browser.

Iterative improvement

Bugs caught in review

Each component went through at least two review rounds before commit. These are the bugs that made it past the first draft.

Critical

CJS bundle broke with .js extension under "type": "module"

Found in: Secondary review

→ Changed fileName to return .cjs for CommonJS format. The extension tells Node.js the module system regardless of "type" in package.json.

Critical

aria-live region never announced dynamic errors

Found in: Accessibility audit

→ Mounted the description span always (empty), controlled visibility with CSS :empty. Live regions must exist before content arrives.

Medium

fullWidth test always passed regardless of implementation

Found in: Code review

→ Replaced toHaveClass("fullWidth") with toHaveAttribute("data-fullwidth"). CSS Module hashing makes class names untestable by string.

Medium

process.env.NODE_ENV left literal in Vite library bundle

Found in: Architecture review

→ Added @rollup/plugin-replace to preserve the expression. Consumer bundler resolves it in their context — same pattern as React itself.

Medium

export type {} syntax error in Button barrel export

Found in: Build verification

→ Exported ButtonProps interface explicitly. export type {} is invalid syntax — the interface must be declared and named.

Low

:empty CSS failed when Prettier added whitespace nodes

Found in: CSS behavior review

→ Wrote the description span in a single line in JSX. React renders undefined as no text node, so :empty works. Empty strings would break it.

Retrospective

What I learned

01

A library's primary customer is not the end user — it's the bundler

Every decision about ESM/CJS formats, sideEffects: false, and peer dependencies exists to make webpack, Vite, and esbuild in the consumer's project work correctly. Getting tree-shaking right requires thinking about how your output is consumed, not just how it's written.

02

Accessibility is architecture, not polish

Decisions like aria-live region mounting strategy, the distinction between aria-disabled and native disabled, and the semantic choice of <hr> vs <div role="separator"> affect how the component behaves for 15% of users. These decisions cannot be added as an afterthought — they change the component API.

03

Documentation written after the fact is always incomplete

Writing ADRs during development forced explicit reasoning at decision time. Decisions that seemed obvious in the moment revealed important context when written down. A decision that takes 30 seconds to make can take 30 minutes to reconstruct two weeks later.
